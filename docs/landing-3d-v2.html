01 / The app

管理实例和使用酒馆，是两件事。

SillyClient 留在控制台里处理安装、端口和日志，SillyTavern 则在单独的窗口中打开。关掉阅读窗口不会停掉服务；要停止实例，回控制台操作。

  • 本地实例 / 远程地址放在一起
  • 安装进度 / 运行日志看得见
  • 返回界面 / 停止服务两回事
手机屏幕可以直接操作
02 / Native runtime

界面共用，底层各自处理。

React 控制台在两个平台保持一致。文件、进程、窗口和运行时交给 Android 的 Kotlin 或 Windows 的 Electron。

运行路径 本机运行
  1. 01
    管理界面

    在这里选版本、创建实例、改端口，也能看安装进度和日志。

  2. 02
    平台端

    Android 用 Kotlin，Windows 用 Electron。两边各自处理文件、进程和窗口。

  3. 03
    Node.js

    运行时跟着安装包走，SillyTavern 作为独立服务启动。

Node.js 已经带上

Android 不要求先装 Termux，Windows 也不借用系统 Node.js。

关窗口，不关服务

离开 SillyTavern 窗口后，实例继续运行。停止操作留在控制台。

前端只有一份

Android 和 Windows 使用同一份 React 构建，平台差异留给 Kotlin 和 Electron。
