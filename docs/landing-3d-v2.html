01 / The app

启动器和酒馆，各做各的事。

控制台负责实例、端口、进程和日志；SillyTavern 在独立的原生窗口中运行。返回控制台时，实例不会被顺手关掉。

  • 本地与远程实例统一管理
  • 实时日志与终端随时查看
  • 返回酒馆 / 停止实例分开控制
屏幕可直接交互
02 / Native runtime

不只是给网页套一层外壳。

共享的 React 控制台只负责发出指令；安装、进程、运行时和阅读窗口，由 Android 或 Windows 的平台端接手。

运行时随应用提供

Android 无需 Termux，Windows 无需另装 Node.js；安装与启动流程由 SillyClient 接管。

控制台与酒馆分开

管理界面和 SillyTavern 使用各自的窗口，退出阅读界面不等于停止实例。

同一套界面，不同平台实现

React 控制台保持一致，Android 的 Kotlin 与 Windows 的 Electron 分别处理原生能力。
